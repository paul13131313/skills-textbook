生成新聞

AIが毎朝・毎夕、新聞を自動生成するサブスク型ニュースサービス（月額$3）
▼

🔀 データの流れ

RSS / Stooq / Open-Meteo→ Cloudflare Workers→ Claude API→ React (PWA)→ 読者
Stripe→ Workers (Webhook)→ Resend→ メール配信
Cloudflare Workersバック
RSSフィード・株価・天気データを収集し、Claude APIに渡して新聞記事を生成するサーバー側の処理を担当
💡 要するに：自分のサーバーを持たずに、間借りで関数だけを動かす仕組み。リクエストが来たときだけ動くので安い。
Claude APIAI
収集したRSSデータを新聞の文体で記事に変換。朝刊・夕刊それぞれ異なるプロンプトで生成
💡 要するに：AIに「こんな文章を書いて」と指示を出して、文章を自動生成してもらうAPI。プログラムからAIを呼び出す窓口。
React (PWA)フロント
新聞の段組みレイアウトを構築。PWA化でスマホのホーム画面に追加可能
💡 要するに：HTMLを効率よく組み立てるための設計図の書き方。部品（コンポーネント）を組み合わせてページを作る。
Stripeインフラ
月額$3のサブスクリプション決済を処理。Webhookで支払い完了をWorkersに通知
💡 要するに：クレジットカード決済を代行してくれるサービス。自分で決済システムを作らなくていい。
ResendAPI
有料会員にメールで新聞を配信
💡 要するに：プログラムからメールを送るためのサービス。Gmailの送信ボタンの代わりにAPIで送る。
RSSAPI
はてブ・artscape・CINRA等のニュースフィードからリアルデータを取得
💡 要するに：Webサイトの更新情報を機械が読みやすい形式で配信する仕組み。新聞の切り抜きサービスのようなもの。
Open-Meteo APIAPI
天気予報データを取得して天気コーナーに表示
Stooq CSVAPI
株価・為替データを取得してティッカー表示

1. AIコンテンツ生成パイプライン

パターン

データ収集→ AI API呼び出し→ 整形・保存→ 表示 / 配信

使用例

  • 生成新聞 — RSS・株価・天気を収集 → Claude APIで新聞記事を生成 → PWAで表示
  • 台灣生成新聞 — 台湾メディア8社をスキャン → 翻訳・要約 → Next.jsで表示
  • 業界紙つくーる — アンケート回答 → web_search付きClaude → メール配信
  • PROTOTIPER — デザインスタイル選択 → Claude Sonnet → ギャラリー
  • 占星新聞 — 天気データ+星座 → Claude Haiku → パーソナライズ占い

変種

リアルタイム推定（MONEY TRAIL：ユーザー入力のたびにAIを呼ぶ）vs バッチ生成（生成新聞：Cronで定時に一括生成）

💡 ポイント：データを集めて → AIに渡して → 結果を見せる、という3ステップが基本形。「何のデータを集めるか」「AIに何を生成させるか」が変わるだけ。

2. カメラ → AI → ビジュアル

パターン

getUserMedia→ MediaPipe / TensorFlow.js→ Canvas / WebGPU描画

使用例

  • AIR SYNTH — 手の検出 → 位置を音程に変換 → シンセ音生成
  • FACE MIRROR — 顔468点検出 → 4種類のエフェクト描画
  • LIQUID WALL — 人物分離 → 背景を流体に置換
  • BODY PARTICLES — 体33点検出 → 20万パーティクルをGPU制御
  • DARK SCANNER — 顔+体+物体の3モデル同時実行 → HUDオーバーレイ

💡 ポイント：映像のフレーム（1秒に30〜60回）ごとにAI推論→描画のループを回す。ループが速いほど滑らか。

3. リアルタイム同期

パターン

ユーザー入力→ 通信層→ 全員に反映

比較表

  • P2P（WebRTC） — SYNC PAD: サーバー不要で直接つなぐ。少人数向き。遅延が最小
  • サーバー経由（WebSocket） — LIVE BOARD: サーバーが中継。多人数対応。自前構築が必要
  • マネージド（Ably） — NUISMO: サービスに丸投げ。Presence(在室管理)付き。開発が楽だが有料

💡 ポイント：「誰と誰をつなぐか」「何人つなぐか」で最適な方式が変わる。少人数ならP2P、多人数ならサーバー経由。

4. Stripe決済フロー

パターン

LP（商品紹介）→ Stripe Checkout→ Webhook受信→ サービス有効化

使用例

  • 生成新聞 — 月額$3サブスクリプション → 新聞閲覧+メール配信の有効化
  • 業界紙つくーる — 決済完了 → アンケート → AI生成 → メール配信開始
  • KANJI ME — 追加機能のアンロック

💡 ポイント：自分で決済画面を作る必要なし。Stripeの画面に飛ばして、「支払い完了」の通知（Webhook）を待つだけ。

5. ブラウザ完結型ツール

パターン

ユーザー操作→ ブラウザAPI→ 結果表示 / エクスポート

使用例

  • FOLDER VISION — File System Access APIでフォルダ読取 → D3.jsで可視化
  • COLOR THIEF — Canvas APIでピクセル解析 → カラーパレット表示
  • VOICE MEMO — Web Speech APIで音声認識 → IndexedDBに保存
  • 走馬灯 SOMATO — Canvas + MediaRecorderで動画生成
  • テープ貼るやつ — Canvas上でテープ配置 → PNG出力

💡 ポイント：サーバーが不要なのでホスティングコストがゼロ。GitHub Pagesに置くだけで動く。ブラウザAPIの組み合わせで十分なユースケースは意外と多い。

6. GPU並列処理

パターン

CPU (初期化)→ GPU転送→ Compute Shader (並列計算)→ Render Pipeline (描画)

使用例

  • PARTICLE STORM — 100万パーティクルの位置計算+描画をすべてGPU上で実行
  • BODY PARTICLES — AIが検出した体の位置に20万パーティクルをGPUで引き寄せる

💡 ポイント：CPUは「1つの複雑な計算が得意」、GPUは「同じ単純な計算を何百万個も同時にやるのが得意」。パーティクルのように同じ計算（位置更新）を大量に繰り返す処理はGPUの独壇場。

7. Cron自動化

パターン

定時トリガー→ データ取得→ 処理・保存→ 表示 / 配信

使用例

  • 台灣生成新聞 — Vercel Cronで毎日定時にニュース生成
  • PROTOTIPER — Vercel Cronで毎日新しいプロトタイプを自動生成
  • 生成新聞 — Cloudflare Workers Cronで朝刊6時・夕刊17時に自動生成

💡 ポイント：人の操作なしで毎日コンテンツが更新される仕組み。一度作ったら放置でOK。ただしAPI料金には注意。

8. モバイルアプリ化

パターン

Web版で検証→ React Native (Expo) で移植→ ネイティブ機能追加

使用例

  • 散歩記録 (Web) → 散歩記録 Expo版 → Sanpos

進化の過程

  • Web版：Geolocation API + Leaflet.jsで基本機能を検証
  • Expo版：expo-locationでバックグラウンドGPSトラッキングを追加
  • Sanpos：Supabaseで認証・DB追加、ソーシャル機能（公開タイムライン）を実装

💡 ポイント：Reactの知識がそのまま使える。Web→ネイティブの移植コストが低い。ネイティブAPI（GPS・カメラ・プッシュ通知）はExpoのライブラリで簡単に追加できる。
